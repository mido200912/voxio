# 📂 هيكل المجلدات وتفاصيل المشروع (Folder Structure & Project Details)

هذا الملف يشرح بالتفصيل هيكل مشروع **VOXIO** وما يفعله كل مجلد تقنياً.

## 📁 Root Directory (المجلد الرئيسي)
يحتوي على المجلدات الأساسية التي تشكل المنصة بالكامل.

- **`backend/`**: السيرفر الرئيسي للمنصة (Main API). يتعامل مع المستخدمين، الشركات، الذكاء الاصطناعي، والتكامل مع المنصات.
- **`frontend/`**: تطبيق الويب الرئيسي (لوحة تحكم المستخدم). مبني باستخدام React + Vite.
- **`admin-backend/`**: سيرفر خاص بلوحة تحكم المدير (Admin API). للتحكم في المستخدمين، الإحصائيات، والرسائل.
- **`admin-dashboard/`**: واجهة لوحة تحكم المدير. مبنية باستخدام React.

---

## 📁 `backend/` (المحرك الخلفي الرئيسي)
هذا هو قلب النظام، حيث تتم معالجة البيانات والذكاء الاصطناعي.

- **`api/`**: يحتوي على تعريفات الـ Vercel Functions (في حال تم استخدامه كـ Serverless).
- **`config/`**: ملفات الإعدادات، مثل الاتصال بـ Firebase (`firebase.js`).
- **`controllers/`**: يحتوي على منطق العمل (Logic). كل ملف هنا يعالج طلبات معينة (مثل `authController.js` للتسجيل والدخول).
- **`middleware/`**: برمجيات وسيطة للتحقق من الصلاحيات (Auth) وحماية المسارات.
- **`models/`**: (إذا كان مستخدماً) تعريفات هياكل البيانات، لكن المشروع يعتمد بشكل أساسي على Firebase Firestore.
- **`routes/`**: تعريف مسارات الـ API (Endpoints) وربطها بالـ Controllers.
- **`utils/`**: وظائف مساعدة عامة (مثل معالجة النصوص أو إرسال الإيميلات).
- **`server.js`**: نقطة انطلاق السيرفر الرئيسي.

---

## 📁 `frontend/` (تطبيق المستخدم)
الواجهة التي يتفاعل معها أصحاب الأعمال.

- **`src/components/`**: المكونات القابلة لإعادة الاستخدام (الأزرار، النماذج، الجداول).
- **`src/context/`**: إدارة الحالة العامة (State Management) مثل اللغة، السمة (Dark/Light)، والمصادقة.
- **`src/layouts/`**: قوالب الصفحات (مثل شكل لوحة التحكم الجانبية).
- **`src/pages/`**: الصفحات الفعلية للتطبيق (الرئيسية، التسجيل، الإعدادات).
- **`src/pages/dashboard/`**: الصفحات الخاصة بداخل لوحة التحكم (التدريب، التكاملات، الإحصائيات).
- **`src/utils/`**: دوال مساعدة خاصة بالواجهة.
- **`public/`**: الملفات الثابتة مثل الصور والأيقونات.

---

## 📁 `admin-backend/` & `admin-dashboard/` (نظام الإدارة)
نظام منفصل تماماً مخصص لمدير المنصة (Super Admin).

- **`admin-backend/server.js`**: يدير عمليات الحظر، تعديل حدود الرسائل، ومشاهدة جميع المستخدمين.
- **`admin-dashboard/src/`**: يحتوي على واجهة بسيطة وسريعة للمدير لمراقبة أداء المنصة.

---

## 📁 المجلدات التقنية الأخرى
- **`.sixth/`**: (محتمل) مجلد خاص بأدوات التطوير أو التحليل.
- **`node_modules/`**: المكتبات الخارجية المثبتة للمشروع.
- **`.env`**: (غير مرفوع) يحتوي على مفاتيح السرية (API Keys) وقواعد البيانات.
