# وثيقة متطلبات المنتج (PRD) - VOXIO

**الإصدار:** 1.0  
**آخر تحديث:** 30 مايو 2026  
**الحالة:** قيد التطوير

---

## 1. ملخص المنتج (Product Summary)

**VOXIO** هي منصة SaaS متكاملة تتيح للشركات إنشاء وكيل ذكاء اصطناعي (AI Agent) خاص بها، وتدريبه على بيانات الشركة، ثم ربطه بعدة قنوات تواصل مثل تيليجرام، واتساب، إنستجرام، فيسبوك ماسنجر، شوبيفاي، وودجيت الموقع الإلكتروني. تهدف المنصة إلى أتمتة خدمة العملاء والمبيعات باستخدام الذكاء الاصطناعي مع دعم كامل للغة العربية.

---

## 2. الرؤية (Vision)

أن تكون المنصة الرائدة في الشرق الأوسط وشمال أفريقيا لتكامل الذكاء الاصطناعي مع قنوات التواصل التجارية، وتمكين آلاف الشركات من أتمتة خدمة العملاء والمبيعات بكفاءة عالية وبتكلفة مناسبة.

---

## 3. المهمة (Mission)

تمكين الشركات من بناء ونشر وكلاء ذكاء اصطناعي مخصصين لعلامتهم التجارية دون الحاجة إلى خبرة تقنية، مع توفير تكامل سلس مع جميع منصات التواصل التي تستخدمها الشركات يوميًا.

---

## 4. الجمهور المستهدف (Target Audience)

| الفئة | الوصف |
|---|---|
| **الشركات الصغيرة والمتوسطة** | شركات تحتاج إلى أتمتة خدمة العملاء بتكلفة منخفضة |
| **المتاجر الإلكترونية** | متاجر شوبيفاي التي تريد بوت مبيعات وخدمة عملاء ذكي |
| **العلامات التجارية الناشئة** | تحتاج إلى تواجد على واتساب وإنستجرام مع ردود آلية |
| **وكلاء التسويق الرقمي** | يديرون حسابات عملاء متعددين ويحتاجون أتمتة |
| **السوق العربي (MENA)** | الشركات الناطقة بالعربية التي تعاني من نقص حلول الذكاء الاصطناعي بالعربية |

## 5. شخصيات المستخدمين (User Personas)

### 5.1 أحمد - صاحب متجر إلكتروني
- **العمر:** 32 سنة  
- **الخلفية:** يملك متجر ملابس على شوبيفاي  
- **المشكلة:** يضيع وقت طويل في الرد على استفسارات العملاء في واتساب وإنستجرام  
- **الاحتياج:** بوت ذكي يرد على العملاء ويحولهم لطلبات شراء  

### 5.2 سارة - مديرة تسويق
- **العمر:** 28 سنة  
- **الخلفية:** تدير حسابات إنستجرام لعدة علامات تجارية  
- **المشكلة:** تحتاج إلى ردود تلقائية على التعليقات والرسائل  
- **الاحتياج:** نظام ردود ذكي يتفهم سياق المحادثة  

### 5.3 محمد - صاحب شركة ناشئة
- **العمر:** 40 سنة  
- **الخلفية:** شركة تقنية ناشئة تحتاج دعم فني 24 ساعة  
- **المشكلة:** لا يستطيع توظيف فريق دعم فني كامل  
- **الاحتياج:** وكيل ذكاء اصطناعي يتعلم من وثائق الشركة ويرد بدقة

## 9. تدفق المستخدم (User Flow)

### 9.1 تدفق مستخدم جديد
```
1. زيارة الموقع التعريفي ← 2. التسجيل (OTP) ← 3. تفعيل الحساب
   ← 4. Onboarding (الملف الشخصي) ← 5. Onboarding (المعرفة)
   ← 6. Onboarding (ربط القنوات) ← 7. لوحة التحكم
```

### 9.2 تدفق تدريب الذكاء الاصطناعي
```
1. رفع الملفات (PDF/DOCX/TXT) ← 2. استخراج النص تلقائيًا
   ← 3. دمج المعرفة مع القاعدة الموجودة ← 4. إضافة تعليمات مخصصة
   ← 5. اختبار النموذج في Model Test ← 6. النشر
```

### 9.3 تدفق المحادثة مع العميل
```
1. يرسل العميل رسالة عبر تيليجرام/واتساب/ودجيت
   ← 2. يصل Webhook إلى الباك إند
   ← 3. يتم إرسال السياق + تاريخ المحادثة إلى OpenRouter
   ← 4. يعود الرد من AI
   ← 5. يتم تحليل الرد (هل يحتوي على SAVE_LEAD, SAVE_ORDER, HUMAN_HANDOFF؟)
   ← 6. يتم تنفيذ الإجراءات المطلوبة
   ← 7. إرسال الرد للعميل
```

## 19. الملاحظات والمخاطر (Notes & Risks)

### المخاطر المعروفة
1. **اعتماد على OpenRouter API** - قد يؤثر توقف الخدمة على أداء المنصة. (الحل: دعم نماذج متعددة)
2. **Webhooks للقنوات الخارجية** - حاجة إلى IP ثابت أو HTTPS موثوق. (الحل: Vercel + تكوين Webhooks)
3. **Puppeteer** - قد تواجه مشاكل مع بعض المواقع التي تمنع التجريف. (الحل: Puppeteer Stealth)
4. **التوسع** - مع زيادة المستخدمين تحتاج MongoDB إلى تحسين الأداء (Indexing, Sharding).
5. **الامتثال القانوني** - خصوصية بيانات العملاء والامتثال لـ GDPR/Saudi PDPL.

### الاعتماديات الخارجية
